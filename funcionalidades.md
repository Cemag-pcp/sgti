Objetivo do projeto: Criar uma plataforma para uma empresa que gerencie os tickets de TI. Tanto infra estrutura quanto desenvolvimento. 

Funcionalidades: 
    1. Login via e-mail para acesso de Supervisão e técnico;
    2. Formulário de abertura de chamado público;
    3. Sidebar dinâmico e responsivo;
    4. Tela para gerenciar os chamados;
        4.1. Cada chamado deverá ter uma tela que será possível informar tempo de execução (podendo ser mais de uma), editar parâmetros de abertura. Podendo também adicionar observações de execução da ordem.
        4.2. Tabela deverá conter flags de prioridade, status.
        4.3. Deve ter campo para delegar aquela ordem de serviço para o exeutor correto.
    5. Deverá ter uma tela para gerenciar os ativos da empresa (notebook, celulares, micros) com funcionalidade para definir o responsável daquele equipamento.
    6. Deverá ter uma tela para configurar usuarios acessar o sistema, tambem para cadastrar usuario para abertura de chamado.
    7. Para abertura de chamado dev solicitar apenas (matrícula + nome) Isso criará um cadastro para aquele usuario posteriomente não precisará preencher tudo novamente.

Stacks: Django, Postgresql, Tailwind.
